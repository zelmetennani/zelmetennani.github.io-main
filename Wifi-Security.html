Wifi security is important. From a security perspective it can often be a weak point. Here are some cool penetration testing tools to prod at your local wireless network security.

 



#1 Using a raspberry pi and external network card

 

Raspi’s are awesome. We are going to use one, in combination with a wifi card that supports monitor mode and packet injection, to test how much info we can gather once we have network access. The Raspi programming is essentially installing a linux distro on an sd card. In this case a RaspberryPi may even be a bit of overkill, in terms of resources. It would be cool to get these tools on an even smaller form factor like some kind of arduino with a on-board packet injection supported wifi chip -- project for another day. 

 

Today we are using these:

 

Step one is downloading Kali Linux and burning it to an sd card. Here is one good tutorial on how do that <link>. Once installed, shove that card in your RasPi and boot it up.  Get into that desktop, connect to wifi, and you are good to go.  

 

 

Once you bask in the glory of your new Kali Linux box you’ll want to connect your external wifi card and get it into monitor mode. 

 

Note: You will need an external wifi card that supports packet injection and monitor mode.

This is my favorite one.

 

Find your wifi card name in iwconfig

 

And set it to monitor mode with these commands:

 

Ifconfig wlan1 down

Iwconfig wlan1 mode monitor 

Ifconfig wlan1 up
